# Adrobot Тестовое задание

Это тестовое задание демонстрирует пример работы с системой рекламного трекера **Keitaro**. Система используется для отслеживания прибыльности рекламных кампаний: она редиректит пользователей согласно правилам и показывает статистику переходов.

---

## Задачи проекта

### 1. Создание кампаний

Функционал “создаватор” кампаний позволяет:

- Создавать кампанию с указанием:
  - Названия кампании
  - Гео-кода страны
  - ID оффера Keitaro
- Настраивать 2 потока:
  1. Георедирект: ловит всех пользователей из указанной страны и редиректит на Google
  2. Поток оффера: редиректит на выбранный оффер
- Автоматически назначать домен, группу и источник (через API Keitaro)

> В первой части проекта используется кеширование неизменяемых данных (домены, офферы, источники, группы, действия потоков), чтобы не перегружать API лишними запросами.

### 2. Редактор существующих кампаний

Редактор позволяет:

- Добавлять и удалять офферы в кампании
- Обновлять данные на фронтенде при изменении в API
- Отправлять изменения в Keitaro по запросу
- Хранить текущие изменения локально на фронтенде до подтверждения

## Установка и запуск

1. Клонируйте репозиторий:

```bash
git clone https://github.com/DmitrySh85/adrobot.git
```
2. Заполните .env по образцу из .env.example 


3. Запустите проект через Docker Compose:
```docker-compose up --build -d```


После запуска:
--

Backend будет доступен на http://localhost:8001
При запуске в production-окружении необходимо настроить nginx.

База данных PostgreSQL работает в отдельном контейнере
